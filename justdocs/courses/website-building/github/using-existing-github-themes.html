    使用既有 GitHub Pages 主題的優點

    • 零或極低設定成本：不用自己寫版型與 CSS
    • 相容 GitHub Pages：官方或主流主題，部署最穩

    Theme 類型說明

    1. Built-in themes（內建主題) > ⚠️ Attention: github 幫你 deploy

      可在 _config.yml 加入

       theme: jekyll-theme-cayman

      index.md 加入

       title: just a test
 description: nothing special

    非內建主題: 有／沒有「Use this template」的差別

    有「Use this template :一鍵建立全新儲存庫

    • 直接套用他人提供的網站骨架／設定

    沒有「Use this template」: 只能用 Fork 或 Clone

    Fork

    • 與原作者 repo 有關聯
    • 需要理解 upstream、同步更新

    Clone

    • 只下載到本機
    • 還要自己建立 GitHub repo、設定 remote、push

    Use this template (UTT)

    Engine（引擎，擇一): Ruby Go Node.js

    Jekyll（Engine Ruby config _config.yml)

    • minimal-mistake starter

    Hugo（Engine Go config config.toml)

    Astro（Engine Node.js config astro.config.mjs )

    • netlify-templates/astro-quickstart
